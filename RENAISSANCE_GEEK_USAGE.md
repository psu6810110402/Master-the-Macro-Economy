# 🎨 Renaissance Geek - คู่มือการใช้งาน (Usage Guide)

เอกสารนี้รวบรวมวิธีการใช้งานเครื่องมือและสกิลต่างๆ จาก **Impeccable** (สำหรับการปรับปรุง Design) และ **code-review-graph** (สำหรับประหยัด Token และการรีวิวโค้ดอย่างมืออาชีพ) ที่ได้ติดตั้งไว้ในโปรเจกต์ของคุณ

---

## 🏗️ 1. Code Review Graph (ตัวช่วยรีวิวโค้ด & ประหยัด Token)

ระบบนี้จะสร้างแผนผังโค้ด (AST / Knowledge Graph) ช่วยให้ AI ไม่ต้องประมวลผลอ่านโค้ดทั้งโปรเจกต์ใหม่ทุกครั้งในแต่ละ Prompt ทำให้ประหยัด Token ได้ถึง 6.8 เท่าและวิเคราะห์ความเชื่อมโยงของไฟล์ได้แม่นยำขึ้น

**คำสั่งใน Terminal (CLI):**
- `code-review-graph build` : สร้างกราฟครั้งแรก (ควรทำทุกครั้งที่เริ่มโปรเจกต์ใหม่หรือหลังโคลนโปรเจกต์)
- `code-review-graph update` : อัปเดตกราฟเฉพาะไฟล์ที่มีการเปลี่ยนแปลง (ทำงานเร็วมาก ภายใน < 2 วินาที)
- `code-review-graph status` : ดูสถานะและสถิติปริมาณกราฟข้อมูลโค้ดเบสของคุณ
- `code-review-graph watch` : สั่งให้ระบบรันโหมดติดตามเพื่ออัปเดตกราฟอัตโนมัติเบื้องหลัง เมื่อมีการอัปเดตไฟล์

**คำสั่ง Slash Commands (พิมพ์หน้าแชทบอก AI):**
- `/code-review-graph:build-graph` : ให้ AI อัปเดตหรือสร้างโครงสร้างกราฟโค้ดให้
- `/code-review-graph:review-delta` : ให้ AI ช่วยรีวิวเฉพาะความเปลี่ยนแปลงของโค้ดล่าสุด (ช่วยจำกัดวงการดูโค้ดเฉพาะสิ่งที่เพิ่งแก้ไป)
- `/code-review-graph:review-pr` : ให้ AI รีวิวโค้ดแบบจัดเต็มสำหรับ Pull Request พร้อมโชว์ว่าโค้ดที่แก้กระทบส่วนไหนบ้าง (Blast-radius)

---

## 💅 2. Impeccable Design Skills (ชุดคำสั่งยกระดับ UI/UX)

เวลาที่คุณต้องการให้ AI เข้ามาช่วยปรับแต่งหน้าตาเว็บไซต์หรือแอปพลิเคชัน แทนที่จะสั่งด้วย Prompt ธรรมดา คุณสามารถพิมพ์ `/ชื่อสกิล` ตามด้วยบริเวณที่ต้องการได้เลยครับ ตัวอย่างเช่น `/polish checkout-form` หรือ `/audit header`

### 🛠️ หมวดวิเคราะห์และตรวจจับ (Diagnostic)
- `/audit` : สั่งให้ตรวจสอบคุณภาพงานทางเทคนิคทั้งหมด (เช่น โครงสร้าง Accessibility, Performance, Responsive layout) เพื่อหาจุดบกพร่อง
- `/critique` : ให้ AI รีวิวงานออกแบบในมุมมองของ UX สแกนหาความบกพร่องด้านการสื่ออารมณ์ โครงสร้างข้อมูล และลำดับความสำคัญ พร้อมบอกวิธีแก้ชัดเจน

### ✨ หมวดเพิ่มคุณภาพดราฟท์ (Quality)
- `/normalize` : ปรับดีไซน์ให้ถูกต้องและสอดคล้องกับ Design System มาตรฐานของโปรเจกต์ ลบความไม่สม่ำเสมอ (Inconsistency) ของสไตล์ทิ้ง
- `/polish` : "ขัดเงา" ขั้นสุดท้ายก่อน Deploy ช่วยแก้ปัญหาจุกจิกเล็กๆ เช่น สเปซบางจุด, การพรางตา (Alignment) ที่จะช่วยเปลี่ยนจากงาน "ดี" เป็นงาน "เยี่ยม"
- `/optimize` : เพิ่มประสิทธิภาพหน้า UI ตั้งแต่ความเร็วในการโหลด, ความสมูทของแอนิเมชัน และการลดขนาดการเรนเดอร์ภาพ
- `/harden` : ปรับ UI ให้ "ถึกทน" พร้อมเผชิญโลกความเป็นจริง เช่น จัดการการแสดงผลกรณี Text Overflow, รองรับภาษาอื่นๆ (i18n), Edge cases และ Error states

### 📐 หมวดการดัดแปลง (Adaptation)
- `/clarify` : ช่วยเสกข้อความ (UX Copy) เช่น คำเตือน (Error messages), ฉลาก (Labels) ให้ผู้ใช้อ่านแล้วเก็ตทันที ไม่สับสน
- `/distill` : ตัดทอนความซับซ้อนหรือองค์ประกอบที่ไม่จำเป็นออกไป ทำให้ดีไซน์ดูทรงพลัง ชัดเจน และสะอาดตา
- `/adapt` : ปรับมุมมองการออกแบบหน้าจอเพื่อให้รองรับมือถือ แท็บเล็ต หรือเงื่อนไขหน้าจออื่นๆ ได้อย่างสมบูรณ์แบบ (Responsive Experience)

### 🚀 หมวดลูกเล่นและยกระดับสุนทรียภาพ (Enhancement)
- `/animate` : เติมเต็ม UI ด้วยแอนิเมชันหรือ Micro-interactions ที่มีจุดมุ่งหมาย ไม่น่าเบื่อ และทำให้ผู้ใช้งานลื่นไหล
- `/colorize` : เพิ่มคู่สีและสไตล์สีให้กับ UI ที่ดูจืดชืดเกินไป ช่วยนำสายตาให้น่าดึงดูดและลงตัว
- `/delight` : เติม "ความประทับใจ" (Moments of joy) หรือลูกเล่นเล็กน้อยที่ทำให้ผู้คนจดจำ
- `/arrange` : ขบคิดและปรับปรุงโครงสร้าง Layout, สเปซ (Spacing), ลำดับการมองเห็น (Visual Rhythm) ให้เป็นระเบียบสวยงาม
- `/typeset` : จัดการการใช้ตัวอักษร การเลือกฟอนต์ น้ำหนัก ขนาด และลำดับความสำคัญ ให้น่าอ่านและดูเป็นตัวจริง
- `/onboard` : ออกแบบหรือปรับปรุง Flow การทำงานชิ้นแรก เช่น การสอนผู้ใช้สมัคร (Onboarding), หน้า Empty state ตอนไม่มีข้อมูลข้อมูล
- `/overdrive` (BETA) : ทะลุขีดจำกัดหน้าตาแอป/เว็บแบบเดิมๆ ให้ AI เสกโครงสร้างเทคนิคขั้นสูงล้ำๆ ที่ชวนให้ผู้ใช้อุทาน "ทำได้ยังไง?" (เช่น Shader, ระบบ 3D, Spring Physics, Scroll-driven animation)

### 🎚️ หมวดปรับระดับอารมณ์และสไตล์ (Intensity)
- `/bolder` : ขยายผลรูปแบบงานออกแบบที่เพลย์เซฟและดูน่าเบื่อ ให้มีความโดดเด่น ดุดัน แข็งแรง และมีพลัง (Aggressive)
- `/quieter` : ปรับเบาหรือลดทอนความเข้มข้นของดีไซน์ที่ดูรุนแรงหรือใช้สีเยอะเกินไป ให้ซอฟต์ลง ให้ความรู้สึกผ่อนคลาย แต่ยังคงเสน่ห์ไว้

### ⚙️ หมวดระบบ (System)
- `/teach-impeccable` : ✅ แนะนำให้ใช้คำสั่งนี้ครั้งเดียวครับ! เพื่อป้อนบริบทดีไซน์ของคุณและโปรเจกต์ให้ AI ทำความเข้าใจเป็นฐานราก แล้วเซฟลง Config (`.impeccable.md`) ช่วยให้คำสั่งอื่นๆ ต่อจากนี้ เข้าฟิตกับสไตล์แบรนด์ได้ดีขึ้น
- `/extract` : ดึงเอา Component ใดๆ ที่สร้างไว้มาจัดเก็บรวมเป็น Reusable components และทำเป็น Design Tokens เข้าโปรเจกต์อย่างเป็นหมวดหมู่ 

---

💡 **ทริคแนะนำสุดยอด:**
สกิลส่วนใหญ่ของ Impeccable ออกแบบมาเพื่อเป็นกระบวนการ (Workflow) คุณสามารถใช้คำสั่งประกอบกันได้ เช่น:
1. เรียกใช้ `/audit` >> เพื่อดูว่าหน้าไหนมีปัญหา
2. จากนั้นรัน `/critique` >> เพื่อดูคอมเมนต์ด้าน UX
3. แล้วต่อด้วยสายแก้บักรัวๆ: `/normalize` + `/clarify` + `/typeset`
4. ปิดท้ายด้วย `/bolder` + `/delight` เพื่อสร้างความว้าวหน้าบ้าน
5. ส่งมอบด้วยคุณภาพสูงสุดผ่านคำสั่ง `/polish` + `/harden`
